## 1. 文档概述

### 1.1 背景与现状

股票提醒助手当前版本（v3.8.0）已具备个股实时行情、分时走势图与底部自选股列表功能。现有自选股以**单一扁平列表**形式呈现（截图所示共 21 只股票，按 4 列网格平铺），存在以下痛点：

- **无分组能力**：所有自选股混在一个列表中，用户无法按板块、策略、持仓/观察等维度分类管理。
- **排序固定**：列表顺序无法调整，无法按涨跌幅、自定义优先级等排序。
- **列不可配置**：每只股票仅展示「名称 + 价格 + 涨跌额」，用户无法查看涨跌幅、成交额、自选时间等更多字段，也无法隐藏不关心的字段。
- **看板信息密度低**：随着自选股数量增长，扁平列表难以快速定位目标个股。

### 1.2 目标与价值

| 维度 | 目标 |
|---|---|
| 用户价值 | 让用户按自身投资逻辑组织自选股，提升盯盘效率与信息获取速度 |
| 产品目标 | 自选股管理从"扁平列表"升级为"分组 + 可配置看板"，提升插件留存与使用时长 |
| 业务目标 | 提升插件日活（DAU）与人均自选股数量，为后续智能提醒/AI 诊股提供结构化数据基础 |

### 1.3 非目标（本期不做）

- 不做云端账号同步（本期仅本地存储，云端同步列入后续版本）。
- 不做自选股的自动推荐 / 智能分组。
- 不做行情数据的字段扩展（仅复用现有行情接口字段）。
- 不重构个股详情页与分时图模块。

### 1.4 术语表

| 术语 | 含义 |
|---|---|
| 分组（Group） | 用户创建的自选股集合，如"持仓"、"观察"、"半导体" |
| 看板（Board） | 分组内自选股的展示视图，支持列配置与排序 |
| 自定义列（Custom Column） | 用户可勾选展示/隐藏的数据字段 |
| 默认分组 | 系统自动创建的"全部"分组，不可删除，包含所有自选股 |

## 2. 功能需求与交互流程（核心）

### 2.1 功能架构

```
自选股分组与看板管理
├── 分组管理
│   ├── 新建分组
│   ├── 重命名分组
│   ├── 删除分组
│   ├── 分组排序（拖拽）
│   └── 股票移入/移出分组
├── 看板展示
│   ├── 分组切换（Tab）
│   ├── 列表/网格视图切换
│   └── 自选股卡片渲染
├── 排序
```

### 2.3 详细交互流程

#### 2.3.1 分组管理

**新建分组**

1. 用户在自选股区域顶部点击「+ 新建分组」按钮。
2. 弹出轻量输入框（非模态弹窗，内联展开），输入分组名称（1-12 字符）。
3. 点击确认或回车，创建分组并自动切换到该分组 Tab。
4. 校验：名称非空、不重复（重复时提示"分组名已存在"）、不超过 12 字符。
5. 限制：单个用户分组上限 20 个（含默认"全部"分组）。

**重命名分组**

1. 鼠标悬停分组 Tab，出现「⋯」更多操作入口。
2. 点击「重命名」，Tab 标题变为可编辑输入框，预填原名称。
3. 修改后回车确认 / 失焦保存 / Esc 取消。
4. 校验规则同新建。

**删除分组**

1. 分组 Tab「⋯」→「删除分组」。
2. 二次确认弹窗："删除分组「XX」？组内 N 只股票将移回「全部」分组，不会丢失。"
3. 确认后删除分组；组内股票归入"全部"分组（若同时属于其他分组则保留）。
4. 默认"全部"分组不可删除，操作入口置灰。

**分组排序（拖拽）**

1. 长按分组 Tab 拖拽到目标位置。
2. 拖拽过程中显示占位指示线。
3. 松手后顺序即时保存。"全部"分组默认固定首位，可配置是否允许拖动（默认锁定首位）。

#### 2.3.2 股票移入/移出分组

**移入分组**

1. 在自选股卡片上右键 / 长按 / 点击卡片「⋯」→「移动到分组」。
2. 展示分组列表（含"新建分组"快捷入口），勾选目标分组（支持多选，一只股票可属多个分组）。
3. 确认后即时生效，当前看板刷新。

**批量操作**

1. 看板顶部「编辑」按钮进入多选模式。
2. 勾选多只股票 → 底部操作栏「移动到分组」「移出自选」「取消」。
3. 完成后退出多选模式。

#### 2.3.3 看板展示与视图切换

**分组 Tab 切换**

1. 顶部横向 Tab 栏展示所有分组，当前分组高亮。
2. 点击 Tab 切换，看板内容平滑切换（无刷新闪烁）。
3. Tab 数量超出可视宽度时左右滚动，并显示分组管理入口。

**视图切换**

- **网格视图**（默认，兼容现状）：4 列卡片网格，每卡片展示名称 + 价格 + 涨跌额，背景色标识涨跌（红涨绿跌，与现状一致）。
- **列表视图**：行式布局，每行一只股票，按自定义列配置展示多字段，信息密度更高。

切换按钮位于看板右上角，记忆用户上次选择（按分组独立记忆）。

#### 2.3.4 排序

**按字段排序**

1. 列表视图下，点击列标题切换排序（升序/降序/取消）。
2. 网格视图下，通过看板顶部「排序」下拉选择字段与方向。
3. 可选排序字段：涨跌幅、现价、涨跌额、成交额、成交量、自选时间。
4. 排序状态在分组内记忆。

**自定义拖拽排序**

1. 选择排序方式为"手动排序"后，卡片/行可长按拖拽调整顺序。
2. 拖拽结果即时保存。
3. 切换到字段排序后再切回手动排序，恢复上次手动顺序。

**股票置顶**

1. 卡片「⋯」→「置顶」。
2. 置顶股票固定在分组顶部，带置顶标识图标。
3. 再次点击「取消置顶」恢复原位。

#### 2.3.5 列配置

**勾选展示字段**

1. 看板右上角「列设置」按钮（齿轮图标）打开列配置面板。
2. 面板列出所有可选字段，以复选框勾选。
3. 可选字段见下方数据字段定义（3.1 节）。
4. 至少保留 1 个字段（不允许全部取消）。
5. 配置按分组独立记忆。

**列顺序拖拽（列表视图）**

1. 列配置面板内可拖拽字段调整左右顺序。
2. 或在列表视图表头直接拖拽列标题调整顺序。

#### 2.3.6 添加自选股到分组

1. 个股详情页「加入自选」按钮 → 弹出分组选择（默认勾选当前分组）。
2. 全局搜索股票 → 搜索结果项「+」按钮 → 加入指定分组。
3. 添加成功后轻提示"已加入「XX」分组"。

### 2.4 边界与异常处理

| 场景 | 处理方式 |
|---|---|
| 分组数达上限（20） | "新建分组"置灰，提示"分组已达上限，请先删除无用分组" |
| 分组名为空/超长 | 输入框红框提示，禁止提交 |
| 分组名重复 | 提示"分组名已存在，请更换" |
| 删除含股票的分组 | 二次确认，股票回收到"全部"分组 |
| 分组内股票为空 | 展示空状态插画 + "添加自选股"引导按钮 |
| 拖拽排序时网络/存储失败 | 回滚到操作前顺序，Toast 提示"保存失败，请重试" |
| 存储空间不足（本地） | 提示"本地存储已满，请精简自选股或分组" |
| 行情数据未返回 | 价格/涨跌显示"--"，不阻断看板渲染 |

### 2.5 状态流转

**分组状态**：活跃（正常使用）/ 空分组（无股票，可保留）。

**股票与分组关系**：多对多——一只股票可属于多个分组，"全部"分组始终包含所有自选股（系统视图，非独立存储）。

## 3. 数据字段与接口定义

### 3.1 自选股可配置字段

| 字段 Key | 显示名 | 数据来源 | 默认展示 | 说明 |
|---|---|---|---|---|
| `name` | 股票名称 | 行情接口 | ✅ | 不可取消 |
| `code` | 股票代码 | 行情接口 | ❌ | 如 sz300418 |
| `price` | 现价 | 行情接口 | ✅ | |
| `change` | 涨跌额 | 行情接口 | ✅ | |
| `changePercent` | 涨跌幅 | 行情接口 | ❌ | 百分比 |
| `open` | 今开 | 行情接口 | ❌ | |
| `prevClose` | 昨收 | 行情接口 | ❌ | |
| `high` | 最高 | 行情接口 | ❌ | |
| `low` | 最低 | 行情接口 | ❌ | |
| `volume` | 成交量 | 行情接口 | ❌ | |
| `amount` | 成交额 | 行情接口 | ❌ | |
| `addedAt` | 自选时间 | 本地存储 | ❌ | 加入自选的时间戳 |

### 3.2 本地数据模型

分组与自选股配置存储于浏览器插件本地存储（`chrome.storage.local`）。

**分组数据结构**

```json
{
  "groups": [
    {
      "groupId": "g_1689012345678",
      "name": "持仓",
      "order": 1,
      "isDefault": false,
      "createdAt": 1689012345678,
      "updatedAt": 1689012345678
    }
  ]
}
```

**自选股与分组关系**

```json
{
  "watchlist": [
    {
      "code": "sz300418",
      "name": "昆仑万维",
      "groupIds": ["g_all", "g_1689012345678"],
      "manualOrder": { "g_1689012345678": 0 },
      "pinned": { "g_1689012345678": true },
      "addedAt": 1689012345678
    }
  ]
}
```

**看板配置（按分组）**

```json
{
  "boardConfig": {
    "g_1689012345678": {
      "viewMode": "grid",
      "sortField": "changePercent",
      "sortDirection": "desc",
      "columns": ["name", "price", "change", "changePercent"],
      "columnOrder": ["name", "price", "change", "changePercent"]
    }
  }
}
```

> 说明：`g_all` 为默认"全部"分组的固定 ID，不可删除。

### 3.3 接口定义

#### 3.3.1 行情数据接口（复用现有）

本功能不新增行情接口，复用 v3.8.0 现有行情拉取逻辑。插件按当前可见分组的股票列表批量请求行情数据。

**请求**：现有批量行情接口，入参为股票代码列表。

**响应字段映射**：见 3.1 节字段表。

#### 3.3.2 本地存储读写（插件内部）

| 操作 | 方法 | 说明 |
|---|---|---|
| 读取全部分组 | `chrome.storage.local.get('groups')` | 启动时加载 |
| 保存分组 | `chrome.storage.local.set({groups})` | 增删改后写入 |
| 读取自选股 | `chrome.storage.local.get('watchlist')` | |
| 保存自选股 | `chrome.storage.local.set({watchlist})` | |
| 读取看板配置 | `chrome.storage.local.get('boardConfig')` | |
| 保存看板配置 | `chrome.storage.local.set({boardConfig})` | 列配置/排序变更后写入 |

#### 3.3.3 数据迁移

v3.8.0 既有自选股为扁平列表，升级到 v3.9.0 时需执行一次性迁移：

1. 读取旧版 `watchlist`（股票代码数组）。
2. 自动创建默认"全部"分组（`g_all`）。
3. 将所有既有股票的 `groupIds` 设为 `["g_all"]`。
4. 写入新版数据结构，保留旧字段作为备份（`watchlist_legacy`）。
5. 迁移在插件启动时检测到旧结构时自动执行，对用户无感。

## 4. 非功能需求

### 4.1 性能

| 指标 | 要求 |
|---|---|
| 看板首屏渲染 | 分组切换后 ≤ 300ms 完成渲染（100 只股票内） |
| 拖拽排序响应 | 拖拽跟手延迟 ≤ 50ms，松手保存 ≤ 100ms |
| 行情刷新频率 | 沿用现有频率（盘中），切换分组不触发额外请求 |
| 本地存储写入 | 单次配置变更写入 ≤ 50ms，不阻塞 UI |
| 内存占用 | 分组+自选股配置数据内存占用增量 ≤ 500KB |

**优化策略**：
- 仅请求当前可见分组的行情数据，切组时按需拉取。
- 看板配置变更使用防抖（debounce 200ms）后批量写入存储。
- 虚拟滚动：列表视图自选股超过 50 只时启用虚拟列表，避免全量渲染。

### 4.2 兼容性

| 维度 | 要求 |
|---|---|
| 浏览器 | Chrome 90+、Edge 90+、其他 Chromium 内核浏览器（如 360、QQ 浏览器）最新版 |
| Manifest 版本 | 兼容 Manifest V3 |
| 操作系统 | Windows 10+、macOS 11+、Linux 主流发行版 |
| 屏幕分辨率 | 1280×720 起步，支持高 DPI 缩放 |
| 数据兼容 | 平滑兼容 v3.8.0 自选股数据（见 3.3.3 迁移） |
| 降级 | 不支持 `chrome.storage.local` 时回退到 `localStorage` 并提示 |

### 4.3 安全

| 维度 | 要求 |
|---|---|
| 数据存储 | 自选股配置仅存本地，不上传服务器，不含用户身份信息 |
| 输入校验 | 分组名称做长度与字符校验，防 XSS（转义后存储与渲染） |
| 权限 | 不新增插件权限，沿用现有行情请求域名白名单 |
| 第三方数据 | 行情数据来源（新浪财经）沿用现有授权方式，不二次分发 |

### 4.4 可维护性

- 分组/看板配置逻辑与行情渲染逻辑解耦，便于后续接入云端同步。
- 字段配置表（3.1 节）以配置驱动，新增字段仅需扩展字段表，无需改 UI 逻辑。

## 5. 埋点与上线指标

### 5.1 关键埋点事件

| 事件 ID | 事件名 | 触发时机 | 关键属性 |
|---|---|---|---|
| `watchlist_group_create` | 新建分组 | 用户确认创建分组 | group_count（当前分组总数） |
| `watchlist_group_delete` | 删除分组 | 用户确认删除分组 | stock_count（组内股票数） |
| `watchlist_group_switch` | 切换分组 | 用户点击 Tab 切换 | from_group_type、to_group_type |
| `watchlist_stock_move` | 移动股票 | 股票移入/移出分组 | stock_count、target_group_id |
| `watchlist_sort_change` | 排序变更 | 切换排序字段/方向 | sort_field、sort_direction |
| `watchlist_column_config` | 列配置变更 | 勾选/拖拽列 | columns_selected、column_count |
| `watchlist_view_switch` | 视图切换 | 网格/列表切换 | view_mode |
| `watchlist_drag_sort` | 拖拽排序 | 手动拖拽完成 | group_id |

> 埋点遵循插件现有埋点通道与用户隐私策略，不含股票代码等敏感明细（仅记 count 与类型）。

### 5.2 上线指标

| 指标 | 定义 | 目标值 | 衡量周期 |
|---|---|---|---|
| 分组功能渗透率 | 使用过至少 1 次新建分组的活跃用户 / 活跃用户 | ≥ 30% | 上线后 14 天 |
| 人均分组数 | 有分组行为的用户平均分组数 | ≥ 2.5 | 上线后 14 天 |
| 列配置渗透率 | 使用过列配置的用户 / 活跃用户 | ≥ 20% | 上线后 14 天 |
| 列表视图使用率 | 切换到列表视图的用户 / 活跃用户 | ≥ 25% | 上线后 14 天 |
| 自选股管理留存 | 上线后 7 日仍使用分组功能的用户 / 首日使用用户 | ≥ 50% | 上线后 7 天 |
| 人均自选股数增量 | 上线后人均自选股数较上线前增长 | ≥ 15% | 上线后 30 天 |
| 看板渲染性能达标率 | 分组切换渲染 ≤ 300ms 的会话占比 | ≥ 95% | 上线后持续 |

### 5.3 灰度与发布

1. **灰度策略**：先灰度 10% 用户，观察 3 天分组功能渗透率与崩溃率。
2. **回滚条件**：崩溃率 > 1% 或数据迁移失败率 > 0.5% 时全量回滚。
3. **全量发布**：灰度指标达标后开放全量。
